# Internationalization (i18n)

This project uses `i18next` + `react-i18next` with German (`de`) as the default and French (`fr`) as an alternate language. Language choice is stored in `localStorage` (`lang`) and selected via the navbar toggle.

## Where things live
- Init: `client/src/i18n.js` (sets resources, fallback, persists language).
- Strings: `client/src/locales/de/translation.json`, `client/src/locales/fr/translation.json`.
- Provider: `I18nextProvider` wrapped around the app in `client/src/main.jsx`.
- Language switcher: in `client/src/components/Navbar.jsx`.
- Category name overrides: `categoryNames` section in each locale (used by product filters and category breadcrumbs).
- Admin labels: `admin.*` namespace in the locale files (dashboard + sidebar nav).

## Adding translations for new UI text
1) Pick a key path that groups logically (e.g., `productDetail.tabs.reviews`, `admin.orders.title`).
2) Add the key/value to **both** locale files. Keep keys ASCII and descriptive; reuse existing namespaces when possible.
3) Use `useTranslation` to pull the string in your component:
   ```jsx
   import { useTranslation } from 'react-i18next';
   const { t } = useTranslation();

   // Simple
   <h2>{t('productDetail.tabs.reviews')}</h2>

   // With interpolation / plurals
   <span>{t('cart.total', { total: total.toFixed(2) })}</span>
   <p>{t('productList.resultCount', { count: items.length })}</p>
   ```
   - For pluralizable strings, include both singular/plural forms in the locale (`key` and `key_plural`).
   - For dynamic names (e.g., categories), add an entry under `categoryNames` with a slugged key (e.g., `Accessories` â†’ `"Accessories": "Accessoires"`); `ProductFilters` already looks there.

4) Avoid hardcoded mixed-language fallbacks. If you need a fallback label, prefer `t('path', { defaultValue: 'Your text' })`.

## Common keys already in place
- `status.*` for loading/error UI.
- `nav.*`, `footer.*`, `home.*`, `productList.*`, `filters.*`, `productDetail.*`, `cart.*`, `payment.*`, `checkout.*`, `login.*`, `signup.*`, `orders.*`, `profile.*`, `admin.*`.
- `categoryNames.*` for known category slugs/names.

## Testing tips
- Switch language in the navbar and sanity-check key screens (Home, Product list/detail, Cart, Checkout, Admin dashboard).
- For new categories, add a translation under `categoryNames` to show the localized label in filters and breadcrumbs.
